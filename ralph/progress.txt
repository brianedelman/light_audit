## Codebase Patterns
- Project uses `postgres:18` base image with custom Dockerfile at `compose/production/postgres/Dockerfile` ŌĆö shared by both local and production docker-compose
- Local dev uses `just` recipes; `just test` runs pytest, `just migrate` runs migrations against local postgres on port 5433
- Ruff is primary linter; mypy has internal errors with django-stubs plugin on Python 3.14 (pre-existing issue)
- Migrations excluded from ruff via `extend-exclude` in pyproject.toml
- Pre-existing lint errors in models.py/admin.py/tests.py/views.py ŌĆö do not fix unless part of story scope
- `daphne` must be first in INSTALLED_APPS (before django.contrib apps)
- Test settings use InMemoryChannelLayer; add new WS consumers to `config/routing.py` websocket_urlpatterns
- Local venv at `.venv/`, use `uv pip install` to add packages
- Docker postgres needs `docker compose build postgres && docker compose up -d postgres` after Dockerfile changes
- `.envs/.local/` is gitignored ŌĆö env file changes must be documented in progress, not tracked in git
- R2 storage backend at `light_audit/audit/storage.py` ŌĆö uses S3-compatible API with region="auto"
- Frontend uses Tailwind CSS v4 with `@tailwindcss/postcss` plugin (not `tailwindcss` PostCSS plugin) ŌĆö import via `@import "tailwindcss"` in CSS
- Vitest config separate from vite.config.ts ŌĆö uses `vitest.config.ts` with jsdom env + `src/test-setup.ts` for jest-dom matchers
- Frontend test script: `npm run test` runs `vitest run`; `npm run build` runs `tsc -b && vite build`
- `.gitignore` has global `lib/` rule ŌĆö files in `frontend/src/lib/` require `git add -f`
- Custom User model: email-based (no username field). Create with `User.objects.create_user(email=..., password=...)`
