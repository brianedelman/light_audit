## Codebase Patterns
- Frontend pages go in `frontend/src/pages/`; tests in `frontend/src/__tests__/`
- Mock api module with `vi.mock('../lib/api')` and cast to typed mock for tests
- Mock TanStack Router's `useNavigate` returning `vi.fn()` for navigation assertions
- Use `makeWrapper()` pattern wrapping `QueryClientProvider` + `AuthProvider` for test renders
- Backend docker-compose not available locally; backend tests require `just pytest` with Docker
- Django Ninja schemas support `resolve_<field>` static methods for computed fields
- Use `annotate()` on querysets to avoid N+1 for computed counts
- TanStack Table: `createColumnHelper<T>()` + `getCoreRowModel`/`getSortedRowModel` for sortable tables
- `useParams({ from: '/route/$param' })` for type-safe param extraction in TanStack Router
- Shared/reusable UI components go in `frontend/src/components/`
